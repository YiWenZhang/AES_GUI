# AES GUI 导出说明

本文档记录当前项目的成功导出方式，方便后续调试代码后重新导出软件。

## 目录约定

- 项目根目录：`D:\Desktop\AES\AES_GUI`
- Python 主程序：`D:\Desktop\AES\AES_GUI\AES_GUI\main.py`
- AES DLL：`D:\Desktop\AES\AES_GUI\AES_GUI\AES_DLL.dll`
- 图标 PNG：`D:\Desktop\app设计\aes-logo.png`
- 生成的 ICO：`D:\Desktop\AES\AES_GUI\export\aes-logo.ico`
- 运行时图标资源：`D:\Desktop\AES\AES_GUI\AES_GUI\assets\aes-logo.ico`
- 导出目录：`D:\Desktop\AES\AES_GUI\export`
- 最终主程序：`D:\Desktop\AES\AES_GUI\export\dist\AES_GUI\AES_GUI.exe`
- 管理员助手：`D:\Desktop\AES\AES_GUI\export\dist\AES_GUI\_internal\AES_Admin_Helper.exe`

## 导出目标

主程序保持普通权限运行，这样文件拖拽不会被 Windows 权限隔离阻止。

只有在软件激活、用户注册这类需要写入 `HKEY_LOCAL_MACHINE\SOFTWARE\AES_Tool` 的操作时，才通过 `AES_Admin_Helper.exe` 弹出系统 UAC，由管理员助手完成注册表写入。

用户体验目标：

- 主程序 `AES_GUI.exe` 不弹黑色 CMD 窗口；
- 管理员助手 `AES_Admin_Helper.exe` 不弹黑色 CMD 窗口；
- 需要管理员权限时只弹 Windows 系统 UAC；
- 成功或失败结果都由主程序界面显示；
- 不打包 zip，只生成可运行文件夹。

## 重新导出步骤

以下命令建议在项目根目录执行：

```bat
cd /d D:\Desktop\AES\AES_GUI
```

如果之前已经导出过，可以先手动删除旧目录，避免旧文件干扰：

```bat
rmdir /s /q export\build
rmdir /s /q export\dist
rmdir /s /q export\helper_build
rmdir /s /q export\helper_dist
```

如果 `export\aes-logo.ico` 已经存在，可以直接复用。若更换了图标 PNG，需要重新生成 ICO，并复制到源码资源目录：

```bat
python -c "from PIL import Image; img=Image.open(r'D:\Desktop\app设计\aes-logo.png'); img.save(r'D:\Desktop\AES\AES_GUI\export\aes-logo.ico', sizes=[(256,256),(128,128),(64,64),(32,32),(16,16)])"
copy /Y D:\Desktop\AES\AES_GUI\export\aes-logo.ico D:\Desktop\AES\AES_GUI\AES_GUI\assets\aes-logo.ico
```

### 1. 先导出管理员助手

管理员助手必须先导出，因为主程序打包时要把它一起带进去。

```bat
python -m PyInstaller ^
  --noconfirm ^
  --clean ^
  --windowed ^
  --name AES_Admin_Helper ^
  --icon D:\Desktop\AES\AES_GUI\export\aes-logo.ico ^
  --distpath D:\Desktop\AES\AES_GUI\export\helper_dist ^
  --workpath D:\Desktop\AES\AES_GUI\export\helper_build ^
  --specpath D:\Desktop\AES\AES_GUI\export ^
  --add-binary "D:\Desktop\AES\AES_GUI\AES_GUI\AES_DLL.dll;." ^
  D:\Desktop\AES\AES_GUI\AES_GUI\src\admin_registry_helper.py
```

这里必须使用 `--windowed`，否则双击或 UAC 启动助手时会出现黑色 CMD 窗口。

导出后应生成：

```text
D:\Desktop\AES\AES_GUI\export\helper_dist\AES_Admin_Helper\AES_Admin_Helper.exe
```

### 2. 再导出主程序

```bat
python -m PyInstaller ^
  --noconfirm ^
  --clean ^
  --windowed ^
  --name AES_GUI ^
  --icon D:\Desktop\AES\AES_GUI\export\aes-logo.ico ^
  --distpath D:\Desktop\AES\AES_GUI\export\dist ^
  --workpath D:\Desktop\AES\AES_GUI\export\build ^
  --specpath D:\Desktop\AES\AES_GUI\export ^
  --add-binary "D:\Desktop\AES\AES_GUI\AES_GUI\AES_DLL.dll;." ^
  --add-binary "D:\Desktop\AES\AES_GUI\export\helper_dist\AES_Admin_Helper\AES_Admin_Helper.exe;." ^
  --add-data "D:\Desktop\AES\AES_GUI\AES_GUI\assets\aes-logo.ico;assets" ^
  D:\Desktop\AES\AES_GUI\AES_GUI\main.py
```

导出后主程序位置应为：

```text
D:\Desktop\AES\AES_GUI\export\dist\AES_GUI\AES_GUI.exe
```

最终运行文件夹中至少要有：

```text
D:\Desktop\AES\AES_GUI\export\dist\AES_GUI\AES_GUI.exe
D:\Desktop\AES\AES_GUI\export\dist\AES_GUI\_internal\AES_Admin_Helper.exe
D:\Desktop\AES\AES_GUI\export\dist\AES_GUI\_internal\AES_DLL.dll
D:\Desktop\AES\AES_GUI\export\dist\AES_GUI\_internal\assets\aes-logo.ico
```

## 验证方式

导出完成后，建议这样测试：

1. 普通双击运行 `AES_GUI.exe`，不要用管理员身份启动主程序。
2. 在软件注册页生成注册码，输入错误注册码时应直接提示无效，不应弹 UAC。
3. 输入正确注册码并点击激活，应只弹 Windows UAC，不应出现黑色 CMD 窗口。
4. 同意 UAC 后，主程序应显示激活成功。
5. 注册新用户时，也应只弹 Windows UAC，结果由主程序显示。
6. 激活和用户注册完成后，测试文件拖拽是否正常。

## 注意事项

- 主程序不要整体设置为管理员运行，否则 Windows 会阻止从普通资源管理器拖拽文件到程序窗口。
- `AES_Admin_Helper.exe` 只负责管理员写注册表，不需要用户直接打开。
- 如果激活或用户注册提示 `激活助手无法加载 AES DLL`，优先检查 `_internal\AES_DLL.dll` 是否存在。
- 如果提示找不到管理员助手，检查 `_internal\AES_Admin_Helper.exe` 是否存在。
- 每次修改 `admin_registry_helper.py`、`admin_registry_service.py`、注册页或登录页后，都建议同时重新导出助手和主程序。
